# 5. Justificación de la GUI

## 5.1 Finalidad de la interfaz gráfica

La interfaz gráfica de OptiAcademic tiene como finalidad facilitar la interacción entre los usuarios y el motor de generación de horarios.

El problema de horarios involucra múltiples datos, restricciones y resultados. Por ello, una GUI es necesaria para que el usuario pueda:

- Registrar información académica.
- Revisar datos existentes.
- Ejecutar diagnósticos.
- Generar soluciones.
- Visualizar horarios.
- Comparar alternativas.
- Publicar resultados.
- Guardar horarios finales.

## 5.2 GUI para administrador y coordinador

La interfaz administrativa permite:

- Gestionar cursos, docentes, estudiantes, aulas y secciones.
- Ejecutar diagnóstico CSP.
- Revisar secciones con problemas.
- Generar horarios institucionales.
- Visualizar bloques generados.
- Publicar oferta académica.

Esto facilita la toma de decisiones porque el usuario no necesita interpretar directamente consultas SQL o salidas técnicas del motor CSP.

## 5.3 GUI para estudiante

La interfaz estudiantil permite:

- Visualizar oferta académica publicada.
- Seleccionar cursos.
- Elegir preferencias.
- Generar alternativas.
- Comparar horarios.
- Elegir horario final.
- Exportar PDF.

Esto mejora la experiencia del estudiante porque transforma un problema complejo en un flujo guiado.

## 5.4 Visualización de resultados

La GUI permite representar resultados mediante:

- Tarjetas de métricas.
- Grillas semanales.
- Listas de bloques.
- Badges de estado.
- Acordeones de diagnóstico.
- Mensajes de éxito y error.
- Comparadores de soluciones.
- Exportación en PDF.

## 5.5 Validación operativa

La interfaz ayuda a validar el sistema porque permite observar si:

- Los horarios se generan correctamente.
- Los bloques tienen aula, docente, día y hora.
- Los diagnósticos muestran problemas claros.
- La oferta publicada llega al estudiante.
- El estudiante puede generar un horario sin cruces.

## 5.6 Conclusión

La GUI no es solo una capa visual, sino un componente clave para validar operativamente el sistema. Permite que usuarios no técnicos interactúen con el motor CSP y tomen decisiones basadas en resultados comprensibles.
