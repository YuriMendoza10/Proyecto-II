# Guia De Presentacion Academica

## Narrativa Sugerida

1. Problema: asignacion manual de horarios y conflictos.
2. Solucion: OptiAcademic como plataforma por roles.
3. Arquitectura: React, FastAPI, MySQL y Docker.
4. Algoritmo: CSP para generar opciones validas.
5. Demostracion: login, gestion, generacion y vista estudiante.
6. Sostenibilidad: dashboard ambiental y GreenFrame.
7. Cierre: resultados, limitaciones y siguientes pasos.

## Evidencias A Preparar

- Captura de la arquitectura o diagrama de componentes.
- Swagger con endpoints principales.
- Dashboard administrativo.
- Resultado de generacion/validacion CSP.
- Horario personal del estudiante.
- Dashboard ambiental.
- Ejecucion GreenFrame exitosa y consulta de `/api/v1/sustainability`.

## Lista Previa A La Demo

- Confirmar que no se muestren secretos en pantalla.
- Ejecutar Docker Compose y verificar los tres contenedores.
- Ejecutar migraciones y seed demo.
- Probar credencial `admin@optiacademic.com / admin123`.
- Regenerar reporte GreenFrame solo si el escenario termina exitosamente.
- Mantener una explicacion clara de que el impacto ambiental es estimado.
