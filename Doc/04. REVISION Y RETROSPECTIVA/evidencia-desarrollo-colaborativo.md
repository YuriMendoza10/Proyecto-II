## Evidencias del Proceso

El trabajo colaborativo se evidencia mediante el historial progresivo de Git,
las ramas de integración, la documentación por fases, las pruebas acumuladas y
la consolidación del PMV. El repositorio registra autores técnicos en Git, pero
no se asignan commits específicos a integrantes sin una validación adicional
de identidades y cuentas.

| Evidencia | Descripción | Estado |
|---|---|---|
| Historial de commits | Cambios progresivos de funcionalidad, pruebas, calidad y documentación | Verificable |
| Ramas | Ramas de integración, corrección, respaldo y demostración | Verificable |
| Integración | Merge commit del PR `#1` hacia la línea estable | Verificable |
| Distribución de tareas | Desarrollo, pruebas, documentación, calidad, sustentación y evidencias | Documentada de forma general |
| Documentación por fases | Inicio, planificación, implementación, revisión, lanzamiento y cierre | Verificable en `Doc/` |
| Validación progresiva | SonarQube, OWASP, WCAG, SUS, Vitest, Pytest, Playwright y Docker | Verificable |
| Participación nominal exacta | Requiere contrastar cuentas GitHub con integrantes | Pendiente de validación manual |

## Roles Generales de Colaboración

La colaboración incluye responsabilidades de desarrollo, pruebas,
documentación, calidad, preparación de la sustentación y recopilación de
evidencias. Esta clasificación no pretende atribuir autoría individual sin
evidencia verificable.

## Comandos de Verificación

```bash
git log --oneline --all
git shortlog -sn --all
git branch -a
```

En GitHub también deben revisarse el historial, las ramas, los pull requests y
la pestaña de contribuciones para completar la trazabilidad nominal.
