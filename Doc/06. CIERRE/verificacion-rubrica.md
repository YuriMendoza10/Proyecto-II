# Verificacion De Rubrica

Esta matriz resume la alineacion del proyecto OptiAcademic con la rubrica de desarrollo web responsable y reduccion del impacto ambiental.

| Criterio de la rubrica | Evidencia en el proyecto | Estado |
|---|---|---|
| Analisis del impacto ambiental | Documentos de sensibilizacion ambiental en `Doc/07. OPTIACADEMIC - SOSTENIBILIDAD Y VALIDACION/sostenibilidad/` y capturas del reporte ambiental en `Doc/06. CIERRE/evidencias/`. | Cumple |
| Identificacion de oportunidades de mejora | Documento de identificacion de oportunidades y matriz de optimizaciones en la seccion de sostenibilidad. | Cumple |
| Implementacion de mejoras de sostenibilidad | Documentacion de paginacion, cache, reduccion de solicitudes HTTP, nombres amigables y modo claro/oscuro. | Cumple |
| Aplicacion de tecnicas especificas de optimizacion | Optimizacion CSP, paginacion gradual, cache de catalogos, lazy loading y reduccion de llamadas duplicadas documentadas. | Cumple |
| Validacion de resultados antes/despues | Plan de validacion, Lighthouse y capturas finales en `Doc/06. CIERRE/evidencias/`. La comparacion cuantitativa completa depende de las capturas registradas por el equipo. | Cumple parcialmente |
| Contribucion a la sostenibilidad del software | Reporte ambiental interno, estimacion de CO2, transferencia de datos y tiempos promedio documentados. | Cumple |
| Gestion del repositorio GitHub | Repositorio organizado, documentacion centralizada en `Doc/`, uso de ramas y recomendaciones de commit documentadas. | Cumple |
| Calidad tecnica y profesional de la documentacion | README principal actualizado, estructura 01 a 07, documentacion tecnica, sostenibilidad y evidencias ordenadas. | Cumple |
| Cumplimiento de recursos y actividades propuestas | Se conservan documentos de inicio, planificacion, implementacion, revision, cierre, sostenibilidad, validacion y evidencias. | Cumple |

## Observaciones

- MegaBlog, MERN, Express y MongoDB aparecen solo como referencias pedagogicas de clase cuando corresponde; la arquitectura oficial de OptiAcademic es React, Vite, Tailwind CSS, Axios, FastAPI, SQLAlchemy, Alembic, MySQL y Docker Compose.
- Las evidencias visuales se encuentran en `Doc/06. CIERRE/evidencias/`.
- La validacion Lighthouse debe mantenerse como evidencia manual para la exposicion final.
