# WCAG - Accesibilidad web

Esta carpeta prepara la evaluacion de accesibilidad de OptiAcademic segun criterios WCAG aplicables a una aplicacion React.

## Rutas a evaluar

- `/login`
- `/admin/dashboard`
- `/admin/institutional-csp`
- `/admin/schedule-view`
- `/admin/reports/sustainability`
- `/admin/students`
- `/admin/teachers`

## Criterios principales

- Contraste de colores.
- Navegacion con Tab.
- Navegacion con Shift + Tab.
- Uso de Enter.
- Uso de Space.
- Foco visible.
- Etiquetas en formularios.
- Botones con texto claro o nombre accesible.
- Mensajes de error comprensibles.
- Tablas legibles.
- Modo claro y modo oscuro.
- Compatibilidad con lector de pantalla.

## Herramientas de evidencia

- Chrome DevTools.
- Lighthouse.
- Panel Accessibility.
- Inspeccion manual del DOM.
- Validacion manual de teclado.

## Evidencias pendientes

- Captura de puntaje Lighthouse Accessibility.
- Capturas de foco visible.
- Capturas de formularios con labels.
- Checklist WCAG completado.
- Hallazgos corregidos o justificados.

No se registran puntajes Lighthouse inventados. Los resultados deben capturarse despues de ejecutar la evaluacion real.
