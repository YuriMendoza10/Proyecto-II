# Video Explicativo Final

## Objetivo

Presentar el PMV de OptiAcademic, su problema, arquitectura, funcionamiento y
evidencias de calidad en un video claro de **máximo 5 minutos**.

**Enlace del video: Pendiente de adjuntar.**

## Estructura Recomendada

| Tiempo | Contenido |
|---:|---|
| 0:00-0:30 | Presentación del proyecto y problemática |
| 0:30-1:00 | Stack y arquitectura resumida |
| 1:00-1:35 | Login y dashboard por rol |
| 1:35-2:25 | Docentes, estudiantes, cursos y aulas |
| 2:25-3:25 | Horarios, CSP y vista de horario |
| 3:25-4:25 | SonarQube, testing, OWASP, WCAG y SUS |
| 4:25-5:00 | Repositorio, documentación y cierre |

## Guion Breve

1. Explicar el problema de planificar horarios con múltiples restricciones.
2. Presentar React, FastAPI, MySQL, Docker Compose y el motor CSP.
3. Demostrar autenticación y dashboard.
4. Mostrar catálogos académicos principales.
5. Ejecutar o mostrar el flujo de generación y publicación de horarios.
6. Presentar evidencias reales de calidad integral.
7. Cerrar mostrando el repositorio y la documentación PMBOK.

## Checklist de Grabación

- [ ] Duración total igual o menor a 5 minutos.
- [ ] Aplicación y datos demo preparados antes de grabar.
- [ ] No mostrar contraseñas, tokens ni archivos de entorno.
- [ ] Mostrar frontend, Swagger y evidencia de Docker saludable.
- [ ] Mostrar generación CSP o un horario ya generado.
- [ ] Mostrar métricas reales, sin alterar resultados.
